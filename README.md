# press-lab-critical-thinking

跟 AI 對話用的「六頂思考帽」協議。

讓你跟 AI 討論事情時，不會被反射性樂觀、跳躍結論、事實與直覺混為一談給拖進泥沼。

> 由 Press Lab 整理開源 · CC BY 4.0

---

## 為什麼有這個專案

跟 AI 討論複雜議題時，你大概遇過這些狀況：

- AI 一句話塞五件事，你來不及消化
- 你還在收集資料，AI 已經跳到結論
- 你提出懷疑，AI 反射性說「你的觀察很精準！」然後接著跑下一個結論
- AI 把訓練資料的印象當事實講出來，你不查證根本不會發現
- 一場討論下來資訊很多，但沒有一個結構幫你判斷哪些有用

這些不是 AI 的錯，是「**討論沒有結構**」造成的。人類自己討論複雜議題時也常這樣。

英國思考學者 Edward de Bono 在 1985 年提出 **六頂思考帽**，把思考拆成六種模式，**一次只用一種**，讓討論慢下來、清楚、有層次。

這個專案做的事：**把六頂帽改寫成一份 AI 對話協議**，讓你跟 AI 共用同一套規則。

---

## 適合誰

- 用 AI 做研究、寫稿、規劃的人
- 受夠 AI 反射性樂觀回應的人
- 想跟 AI 進行嚴肅討論而不是聊天的人
- 教思考方法、要給學生範本的老師
- 任何覺得「跟 AI 討論常常一團亂」的人

**不需要先讀過 de Bono 的書**，這份協議從零開始教。

---

## 怎麼用（30 秒上手）

1. 打開 [SKILL.md](./SKILL.md)
2. 全選複製
3. 貼到 AI 對話框（ChatGPT、Claude、Gemini 都可以）
4. 在貼上的內容下面加一句：「**導入這個 skill**」
5. 開始討論

之後 AI 會主動偵測討論時機、提示換帽、限制自己不要亂下結論。

---

## 六頂帽快速理解

| 帽 | 一句話 |
|---|---|
| ⚪ 白 | 客觀事實 |
| 🔴 紅 | 直覺情緒（不需理由） |
| 🟡 黃 | 找好處（要有理由） |
| ⚫ 黑 | 找壞處（要有理由） |
| 🟢 綠 | 發想新可能 |
| 🔵 藍 | 管理討論流程 |

詳細用法、為什麼這樣設計、AI 該遵守什麼規則，全部寫在 [SKILL.md](./SKILL.md)。

## 跟原版六頂帽的差別

de Bono 的原版設計是給**團體會議**用的。這份協議改寫成**人類 + AI 兩人對話**用，主要調整：

- 紅帽改為人類專屬（AI 沒有真情緒）
- 黃帽預設不主動戴（AI 天然過度樂觀）
- 加入「AI 一句話不塞太多」的限制
- 加入觸發機制（AI 主動偵測該不該戴帽子）
- 切換時必須明確宣告（避免 AI 偷渡）

## 未來方向（Roadmap）

這份協議目前是 v1.0，已經可用。後續想做的事：

**Prompt 範例庫**
針對每一頂帽子，整理實測有效的 System Prompt 或 User Prompt 範本，讓使用者可以直接套用。

**不同場景的帽子組合**
v1.0 提供的是預設順序，但實務上不同情境需要不同組合。例如「危機處理」可能是黑 → 白 → 藍，「產品創新」可能是綠 → 黃 → 黑。整理常見場景的推薦組合。

**AI 的反向制衡**
目前協議假設人類是主席，但當人類戴紅帽過久、決策失焦時，AI 該如何溫柔地提醒回歸理性？這需要設計一套「反向觸發」機制。

歡迎針對任一方向開 Issue 討論，或直接 Pull Request。
